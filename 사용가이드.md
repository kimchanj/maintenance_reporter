# 유지보수 보고서 작성 프로그램 사용가이드

## 1. 프로그램 개요
- 엑셀 유지보수 내역을 읽어 워드 템플릿에 자동 매핑하여 보고서를 생성합니다.
- 생성 직후 엑셀과 워드 결과를 교차검증합니다.
- `진행상태`가 `완료`인 데이터만 반영합니다.

## 2. 준비 파일
- 엑셀 파일: `*.xlsx`
- 워드 템플릿 파일: `*.docx`
- 지원 템플릿 유형
1. 행 확장형 템플릿
: 각 데이터 건을 표의 행으로 추가하여 작성
2. 단일 셀 누적형 템플릿 (VPN 작업내역서 형식)
: 한 셀 안에 여러 건을 순차 누적하여 작성

## 3. 실행 방법
1. `MaintenanceReporter.exe` 실행
2. `엑셀` 버튼으로 입력 엑셀 파일 선택
3. `워드` 버튼으로 템플릿 파일 선택
4. `작성` 버튼 클릭

## 4. 데이터 처리 규칙
1. 템플릿의 `{{항목}}` placeholder를 추출
2. 엑셀 헤더에서 이름이 일치하는 항목만 매핑
3. `진행상태=완료` 행만 필터링
4. `처리일자` 기준 오름차순 정렬
5. 템플릿 유형에 맞춰 문서 작성
6. 작성 완료 후 교차검증

## 5. 결과 파일
- 저장 위치: 선택한 워드 템플릿 파일과 같은 폴더
- 저장 이름: `원본템플릿명_작성완료.docx`

## 6. 완료 메시지 의미
- `교차검증 통과: N건`
: 엑셀에서 반영된 건수(N건)와 워드 결과가 내용/건수 기준으로 일치함을 의미합니다.

## 7. 자주 발생하는 오류와 조치
1. `유효한 엑셀/워드 파일을 선택해주세요`
: 파일 경로가 잘못되었거나 파일이 이동/삭제됨

2. `Missing required Excel column for filtering: 진행상태`
: 엑셀 헤더에 `진행상태` 컬럼이 없음

3. `No rows with 진행상태=완료 found in Excel file`
: 완료 상태 데이터가 없음

4. `Cross validation failed ...`
: 워드 작성 결과가 엑셀 데이터와 일치하지 않음
: 템플릿 placeholder 형식 또는 수작업 수정 여부 확인 필요

## 8. 입력 데이터 권장사항
- 엑셀 첫 20행 안에 헤더 행이 있도록 구성
- 헤더명은 템플릿 placeholder와 동일하게 사용
- 날짜는 가능한 `YYYY-MM-DD` 형식 권장
- 템플릿은 `{{항목명}}` 형식 유지

## 9. 템플릿 작성 팁
- 행 확장형: 데이터 1건이 들어갈 placeholder 행 1개를 준비
- 단일 셀 누적형: 반복 블록 1개를 작성하면 프로그램이 건수만큼 누적 생성

## 10. 배포 시 전달 항목
- `MaintenanceReporter.exe`
- 템플릿 파일(`.docx`)
- 본 사용가이드(`사용가이드.md`)
