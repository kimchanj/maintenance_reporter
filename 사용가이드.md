# 유지보수 보고서 작성 프로그램 사용가이드

## 1. 프로그램 개요
- 유지보수 엑셀에서 완료된 작업을 읽어 VPN 작업내역서를 생성합니다.
- 사용자가 Word에서 VPN 작업내역을 수정한 뒤, 수정본을 기준으로 일반 유지보수 내역서를 생성합니다.
- 일반 유지보수 내역서는 VPN 작업내역의 각 블록에서 첫 줄의 날짜와 둘째 줄의 내용을 사용합니다.

## 2. 준비 파일
- 유지보수 엑셀 파일: `*.xlsx`
- VPN 템플릿 파일: `*.docx`
- 일반 유지보수 템플릿 파일: `*.docx`

## 3. 작업 순서
### 1단계. VPN 내역서 생성
1. 프로그램을 실행합니다.
2. `유지보수 엑셀` 파일을 선택합니다.
3. `VPN 템플릿` 파일을 선택합니다.
4. `VPN 내역서 생성` 버튼을 클릭합니다.
5. 템플릿 폴더에 `원본명_작성완료.docx` 파일이 생성됩니다.

### 2단계. VPN 수정 후 일반 내역서 생성
1. 생성된 VPN 내역서를 Word에서 엽니다.
2. `작업내역` 영역의 내용을 최종본으로 수정합니다.
3. 프로그램에서 `수정된 VPN 내역서` 파일을 선택합니다.
4. `일반 템플릿` 파일을 선택합니다.
5. `일반 내역서 생성` 버튼을 클릭합니다.
6. 일반 템플릿 폴더에 `원본명_작성완료.docx` 파일이 생성됩니다.

## 4. 엑셀 처리 규칙
- `진행상태=완료` 데이터만 반영합니다.
- `진행상태` 컬럼이 없으면 오류가 발생합니다.
- `처리일자` 기준으로 오름차순 정렬합니다.
- 템플릿 placeholder와 일치하는 엑셀 헤더만 매핑합니다.

## 5. VPN 수정본 파싱 규칙
- `작업내역` 셀에서 각 작업 블록은 빈 줄로 구분합니다.
- 각 작업 블록은 최소 2줄이어야 합니다.
- 첫 줄: 날짜가 포함된 메타 정보
- 둘째 줄: 일반 유지보수 내역서의 내용으로 사용
- 셋째 줄 이상이 있어도 일반 유지보수 내역서에는 둘째 줄만 반영합니다.
- 일반 유지보수 내역서의 `- 처리 완료` 문구는 자동으로 작성됩니다.

## 6. 검증
- 1단계: 엑셀 데이터와 생성된 VPN 내역서를 비교합니다.
- 2단계: 수정된 VPN 내역서에서 파싱한 데이터와 생성된 일반 내역서를 비교합니다.
- 검증이 실패하면 저장 중 오류 메시지가 표시됩니다.

## 7. 자주 발생하는 오류
1. `유효한 ... 파일을 선택해 주세요`
- 파일이 없거나 경로가 잘못된 경우입니다.

2. `Missing required Excel column for filtering: 진행상태`
- 엑셀 헤더에 `진행상태` 컬럼이 없습니다.

3. `No rows with 진행상태=완료 found in the Excel file.`
- 완료 상태 데이터가 없습니다.

4. `Could not parse any general-report rows from the VPN document.`
- 수정된 VPN 문서의 `작업내역` 블록이 2줄 구조를 만족하지 않거나 날짜를 찾지 못한 경우입니다.

## 8. 입력 권장사항
- VPN 수정 시 `작업내역` 표 구조는 변경하지 않는 것을 권장합니다.
- 각 작업 항목은 빈 줄로 구분해 주세요.
- 각 작업 항목의 첫 줄에는 날짜를 유지해 주세요.
- 일반 유지보수 내역으로 넘길 문장은 둘째 줄에 작성해 주세요.
